Carpet Feature Requests

As Carpet is being looked at (and used?) by more and more people, they express more and more wishes about things they miss in Carpet. Some of these can be quickly fulfilled, others will take months to implement. Here is a collection of these, so that people know we don't ignore them.

Small Stuff

  • Find a nice Carpet logo. Done. (Well, it's ugly.)
  • Add Doxygen tags. In progress. (Stalled.)
  • Compile and run on a few strange supercomputer architectures. Done.
  • Adapt CarpetSlab to new hyperslabbing interface. Done.
  • Slowly create many test cases for Carpet. In progress.

Heavier Stuff

  • Compile Carpet and PUGH together. Done.
  • Don't restrict all grid functions. Done.
  • Make CarpetInterp interpolate in time as well. Done.
  • Design and write a nicer interface to specify fine grid locations. Done.
  • Introduce the "convergence level" parameter. Done.
  • Do not enable all storage all the time. Done.

Good Stuff

  • Checkpointing and Restart. Done. (Almost.)
  • Parallel performance. In progress.
  • Elliptic solvers. Missing.

Stuff that would be nice, but won't be needed in the near future

  • Cell centring
  • Improve the A of the AMR algorithm
